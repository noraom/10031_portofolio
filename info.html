Oblig 2

We started by downloading the poster and using Adobe Illustrator to extract the svg elements. We used the svgomg to make the file smaller, and more readable. From there we added the code to our index.html, and ensured the right were right. Here we needed to rewrite some class names, so the rules didn't apply for any other svgs than themself. Then we implemented the design and layout with HTML and CSS, stating with implementing the 375px screen and later the larger screen. Some elements change in size, were it's natrual, others have the same sizes. The positioning might not 100% similar to the poster but is mostly the same. It's only small differences that does not take away the meaning of the poster

All the animations are inspired by Lefties lecture about svg files, specific page 54- 68. We wanted the animations to feel natural and be coherent with the design. Therefore, we animated the extracted svg elements with their normal state (in the real world) in mind. Stars: have a Smil animation that transform them to rotate, this is done to simulate the flickering night sky with blinking and shifting stars. I also added a keyframe animation in css to animate the opacity. Together this makes them rotate and flicker, which we thought mimics a shining star in the sky. Moon: Here we added a Smil path animation to make the moon resemble rise and set. This path was made using a vertical path.

Car: We wanted the car to feel like it floats in space. To act like it was taking off in zero gravity we added a path animations using Smil and made a curving path for the car to float and added «repeatCount=indefinite» to make it go on forever. Green figures: Because all the other animations were at the top of the poster, we wanted to even it out and make some of the bottom elements more interesting. Therefore, we added a @keyframe animation to css and made the green figures shake slightly. The figures resemble co2 clouds with angry faces and the shaking just made it even more clear that they are angry and not friendly. We think the animation succeeds to clarifying the message. The animations make the poster feel more alive and interesting. We think the animations contributes to emphasize the message.

Here I think we had done a good job in the original submission. I worked on group with Viktoria Brodshaug Langaas on this assignment. The goal was to recreate the poster we were provided with in the assignment. Here we did add CSS in the case of styling, responsiveness and animations. The assignment had a list of things we had to incorporate into the CSS. I think our original submission was close to what was expected in the assignment. Therefore, I only made minor alternations. We got some feedback, and after what I remember we missed CSS drawing in the original. I added one of the pupils of the eyes to the globe by CSS drawing. The live link version we originally delivered did not include the background, so I added that and made it responsive. The responsiveness is as I can get it within this time window. I also changed the animation of the moon by showing the overflow of the container, that way the moon did not disappear in the middle of the animation. I tried to make some small adjustments to the text so that it would fit the globe a bit better and have better responsiveness. If I would have more time and resources, I would have worked on getting it more responsive and exact like the original poster provided. There are some differences like font style and a bit of positioning that is different, but overall, I am happy with the result!

On this oblig I worked together with Stian Buhayan Gabrielsen and Håkon Andreas Sæbjørnsen. We chose UN sustainable goal number 15 – Life On Land. As a group we decided this would be an interesting and relevant goal for this circumstance. We started by briefly sketching out some ideas about what story we wanted to tell. First, we used sketches and brainstorming to come up with ideas quick, that we could further develop later in the process.

Our story landed on a group of friends heading to vacation, on their way they see different environments along the way. Although they see out of the car, they are mostly focused on getting to their destination. In the car the radio is playing, and a radio host starts talking about different things to put to consideration when being a tourist this summer. The group drive by different kinds of environments, corresponding to the things the radio host tries to communicate. This puts everything into perspective for the friend group, because when they arrive to their destination, the environment is gorgeous. Therefore, they understand the importance of being a thoughtful tourist and leaving the nature the way you found it.

The layout and theme were sketched out in illustrator. There we extracted the svg's we needed to make a cohesive design. Some of the components is downloaded from Adobe Stock (royalty free designs), but it is mostly conducted by us. After the svg components where created, we mocked up a storyboard with all the different scenes and how we ideally wanted it to look. This gave us an overview of the story and components while coding. We found it easiest to create a long svg with the corresponding parts of the different scenes. It made a lot more effective in the code and the overall look was a bit cleaner in our case.

The script works like this. We started by selecting an element called ".palm-view" and all elements with class ".palm-section". Then added a wheel event listener to the window so we can determine the direction of the scroll. After we made the scroll go horizontally. Created an Intersection Observer to track visibility changes of sections when one is in view, change background color based on section class. Under that is a code which resets opacity of child element when not in view. Last is code which Observe visibility changes for each section.

The first thing I started with changing was the text to the story. The feedback was clear, it needed a bit more context and be more cohesive with the design. I think I succeeded with clarifying the message. The css needed a lot of work. In the end I ended up changing every value in the css to view width (vw). This made a lot more responsive on its own, but I tweaked more on the values. I needed to perfect the placements of many of the elements as told, know the placement of the foreground, the animals and bushes makes much more sense. I also did some other minor changes like font family and hover effect on the goal 15 logo in the end.

On the first feedback, we were told to change some colors to make it clearer what svg should represent what. The color choices now make the story more understandable and cohesive. I added Alt text to the images were added to make it validated code. We got feedback that is was a little unclear how to know that it was a radio voice. Therefore, I drew a radio.svg. The text is now displayed on the radio. To make it even more obvious that there is a person talking, I added a talk indicator to the radio to symbolize talking. I fixed the position on the textbox, so it doesn’t move with the palms but changes text so that it is cohesive with the scene.

If I had more time and resources to do it again, I would make some changes and attributions to the code and project. I would like to make it more responsive and have better feel on Phone/tablet view. I did my best with this time, but it could be better. To view the phone/tablet correct you must toggle to horizontal mode. I would also have perfected the script, so it is a bit easier to scroll. The animations should also have a feature where u can toggle them off, but I did not get time to finish this. Overall, I think I made good improvements to the oblig. Now the story serves a clearer purpose, and it works better than it used to.
